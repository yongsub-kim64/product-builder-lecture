# AGENTS.md

## Content Publication Rules

- Before any content update, read `CONTENT-SOP.md`. If a future `CONTENT-PUBLICATION-HARNESS.md` exists, read that as well.
- Log / Insight / Project uploads must follow the publication harness in `CONTENT-SOP.md`.
- Do not invent a new article from scratch. Preserve the experience, facts, judgment, and conclusion in the provided draft.
- Separate SEO working metadata from public body content. SEO briefs, title candidates, slug candidates, and quality-check notes are not public article copy.
- If the draft does not meet the quality gate, do not commit it. Report the specific items that need strengthening.
- Do not add sensitive information such as API keys, tokens, secrets, account identifiers, or private asset details.
- Do not change routes, layouts, components, deployment settings, or repository configuration for content-only work unless explicitly requested.
- Run `npm run build` after content or documentation changes and report the result.
