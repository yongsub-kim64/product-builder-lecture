## 오늘 확인한 것

Notebook을 처음 세팅하면서 중요한 점을 확인했다.

Notebook은 실시간 웹 검색 도구가 아니다. 사용자가 넣어준 소스를 기준으로 답변하고 판단을 보조하는 소스 기반 작업 공간이다.

따라서 Notebook의 품질은 AI 자체보다, 어떤 소스를 어떤 구조로 넣느냐에 크게 좌우된다.

## Notebook의 적절한 역할

철부지닷컴 운영에서 Notebook은 글을 대신 쓰는 도구라기보다, 운영 기준을 기억하고 정합성을 검토하는 장치에 가깝다.

이번 테스트에서 Notebook은 다음 역할에 적합하다는 점을 확인했다.

- Log / Insight / Project / Board 구분
- AI 콘텐츠 비서와 InsightReport 혼동 방지
- 메타철부지, Claude Code, Codex 역할 분담 정리
- Claude Code용 바이브코딩 지시문 생성
- Codex용 점검 체크리스트 생성
- 새 글이나 프로젝트 기획안의 기준 정합성 검토

## 잘못 사용할 수 있는 방식

공개 웹페이지 URL을 많이 넣는 방식은 효율적이지 않다.

사이트 주소를 소스로 추가하면 실시간으로 최신 사이트를 검색하는 것이 아니라, 추가 시점의 정보를 바탕으로 판단하게 된다.

따라서 chulbuji.com 공개 페이지는 상시 소스로 관리하기보다, 특정 시점에 문구나 구조를 검수할 때만 임시 스냅샷처럼 활용하는 것이 적절하다.

기본 소스는 공개 URL보다 운영 기준 문서가 되어야 한다.

## 새로 정리한 운영 기준

이번 테스트를 통해 Codex의 역할도 일부 보완했다.

기존에는 Codex를 읽기 전용 점검 도구로만 정의했지만, Claude Code 토큰 제한이 걸리는 경우에는 예외가 필요하다.

앞으로 Codex는 기본적으로 읽기 전용 점검을 담당하되, Claude Code 사용이 어려운 경우에는 구조 변경이 없는 콘텐츠 반영 작업에 한해 제한적으로 활용할 수 있다.

단, Codex가 새 글을 창작하거나 사이트 구조, 라우팅, 컴포넌트, 디자인을 임의로 변경해서는 안 된다. 메타철부지 또는 사용자가 작성한 원고를 기존 파일 형식에 맞게 반영하는 수준으로 제한한다.

## 다음 활용 방향

앞으로 Notebook은 철부지닷컴의 운영 기억 장치로 활용한다.

새로운 Log, Insight, Project 글을 작성하기 전에는 Notebook을 통해 기존 기준과 충돌하는 부분이 없는지 점검한다.

또한 반복되는 작업은 Notebook을 통해 SOP 초안으로 정리하고, Claude Code와 Codex에게 전달할 지시문과 체크리스트를 생성하는 방식으로 활용한다.

오늘의 결론은 분명하다.

Notebook에는 많은 자료를 넣는 것보다, 판단 기준이 되는 문서를 정확히 넣는 것이 더 중요하다.
