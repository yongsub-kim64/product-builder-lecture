날짜: 2026-04-10

어제 저녁이었다. 두 시간 남짓한 시간 동안 KIS Open API 기반 자동매매 시스템 11개 항목이 정비됐다.

내가 한 일은 코드를 고치는 게 아니었다. 뭐가 문제인지 짚고, 어떤 방향으로 가야 할지 판단하는 것이었다. 실제 분석과 수정은 메타철부지와 개발 전문 AI가 맡았다. 생각보다 빠르게 끝났다.

## 이번에 손댄 것들

**안정성부터 잡았다.**

예수금 필드가 잘못된 값을 참조하고 있다는 걸 확인했다. 문제를 던지니 AI가 원인을 찾고 교체했다. API 호출 실패 시 그냥 멈추던 구조에 재시도 로직이 붙었고, 중복 매수와 중복 기동 방지 처리도 이번에 제대로 들어갔다. hold_days 계산 버그, IS_PAPER 안전장치까지. 하나씩 짚어가면서 나온 결과였다.

**전략도 다듬었다.**

RSI를 Wilder 방식으로 바꾸고 범위를 40~65로 조정했다. MA20 이격도는 5% 이내로 제한을 걸었고, 투자 비중은 20%에 최대 100만 원까지 확대했다. 트레일링 스탑도 처음 도입했다. 방향은 내가 잡았고, 구현은 AI가 했다.

**운영 구조도 정리했다.**

앞으로는 역할을 명확히 나눈다. 방향 판단과 현황 해석은 메타철부지와, 실제 코드 작업은 개발 전문 AI와. 어제 저녁이 그 구조의 첫 번째 실전이었다. 코딩 실력 없이도, 두 시간 안에 시스템이 이만큼 바뀔 수 있다는 걸 직접 확인했다.
