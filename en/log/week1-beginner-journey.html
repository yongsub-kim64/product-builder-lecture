üí• Week 0: The Infinite Loop of Hell

Problem 1: The links keep breaking

Situation:

  • Click "Log" on the homepage ‚Üí 404 Error
  • Click "Home" on the log page ‚Üí Another 404 Error

My attempts:

<!-- I fixed it like this... -->
<a href="log.html">Log</a>

<!-- ...and now this one breaks. -->
<a href="/log.html">Log</a>

<!-- If I change it back... -->
<a href="./log.html">Log</a>

Result: Never-ending problem.

I asked Claude:

Me: "My links keep breaking. Is it log.html or /log.html?"
Claude: "Let me explain the difference between absolute and relative paths..."

10 minutes later, I finally got it:

  • log.html = Look in the same folder.
  • /log.html = Look from the root directory.
  • ./log.html = Look in the current folder.

What I learned: Asking "What's the difference between log.html and /log.html?" is a much better question than just saying "It doesn't work."

Problem 2: The hell of language switching

Situation:

Click "English" on the Korean page ‚Üí Works fine.
Click "ÌïúÍµ≠Ïñ¥" on the English page ‚Üí 404.

The magic of fixing one thing and breaking another:

// Attempt 1

  window.location.href = '/en/index.html';  // KO‚ÜíEN OK!
}
// Result: EN‚ÜíKO 404 üíÄ

// Attempt 2
function switchLanguage() {
  window.location.href = 'index.html';  // EN‚ÜíKO OK!
}
// Result: KO‚ÜíEN 404 üíÄ

// Attempt 3: SOS to Claude
Me: "When I fix one, the other breaks... What should I do?"

Claude's solution:

function switchLanguage() {
  const currentPath = window.location.pathname;
  
  if (currentPath.includes('/en/')) {
    // English ‚Üí Korean
    window.location.href = currentPath.replace('/en/', '/');
  } else {
    // Korean ‚Üí English
    window.location.href = '/en' + currentPath;
  }
}

What amazed me:

  • I learned for the first time that window.location.pathname gives you the current path.
  • I also learned that includes() can check for '/en/'.
  • And that you can change the path with replace()...

What I learned: Instead of just copy-pasting code, I understood it by asking about each line. Now I can solve similar problems myself next time.

Day 1 Afternoon: Starting Vibe Coding

What is Vibe Coding?

The act of building something by grasping the core concepts and collaborating with AI, even without a perfect understanding of the code.

A practical example:

Claude generated 376 lines of mobile-improvements.css.

What I did:

/* 1. I understood this */
@media (max-width: 768px) {  /* Apply only on mobile screens */
  
/* 2. I understood this too */  
  min-height: 44px;  /* Minimum touch area size */

/* 3. I didn't know this, so I asked */
  padding: env(safe-area-inset-top);  /* What's this? */
}

Claude's explanation:

"It's to display content while avoiding the iPhone's notch (the black area at the top). Without it, buttons might get hidden by the notch."

The Vibe Point:

  • You don't need to understand all 376 lines.
  • Understanding the key 10 lines is enough.
  • For the rest, just know "Oh, this is for the notch," and move on.
  • If you need to know more later, you can just ask again.
